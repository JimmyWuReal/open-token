# Agent Instructions

- After making any repository change, commit the change and push the current branch before ending the task, unless the user explicitly asks not to.
- Before committing, inspect `git status` and review the diff for the files you changed.
- Keep commits focused and use clear commit messages that describe the completed change.
- Do not include unrelated user changes in a commit unless the user explicitly asks you to include them.
- If a commit or push fails, report the failure and the exact follow-up needed.
